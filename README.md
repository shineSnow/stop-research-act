# Stop-Research-Act

## AI Agent 目标导向问题解决 SOP

一个让 AI Agent（Claude Code、Cursor、Codex 等）在遇到错误时**不再盲目重试、不再偏离目标**的技能文件。

### 解决的问题

两个痛点：

1. **失败 → 盲目重试循环**：AI 遇到报错就换方案，20 分钟改了无数配置，环境越来越乱
2. **迷失目标**：深入调试某个工具后忘记了最初用户要的是什么。研究了半天 Chrome CDP 协议，忘了用户只是想搜个岗位

**Stop-Research-Act 用结构化的循环打断这两个模式。**

### 核心理念

```
始终记住目标。偏离目标是最大的失败。
失败后，不诊断就重试 = 浪费。
```

### 完整循环

```
PLAN（制定计划）→ 执行 → 失败 → STOP（诊断）→ RESEARCH（搜索）
→ REVISE（修正计划，检查是否偏离目标）→ EXECUTE（执行）→ CHECK（验证+目标对齐）
```

| 步骤 | 做什么 | 核心自检 |
|------|--------|---------|
| **PLAN** | 制定目标、分解步骤、选工具、设成功标准 | 我清楚要给用户交付什么吗？ |
| **STOP** | 读错误信息、分类(A/B/C)、验证依赖链 | 我在诊断还是在试探？ |
| **RESEARCH** | 查文档、搜索社区 | 我的用法和官方一致吗？ |
| **REVISE** | 更新假设、修正计划、**强制检查是否偏离目标** | 我还在朝着原始目标走吗？ |
| **EXECUTE** | 一次改一个变量，立即验证 | 我改了几个变量？ |
| **CHECK** | 验证结果 + 目标对齐 | 修正成功了吗？目标还在吗？ |

### 六条铁律

1. **始终记住目标，每次修正后检查是否偏离**
2. 失败后禁止不诊断就重试
3. 不确定根因必须先搜索，禁止猜测
4. 同一方向失败 3 次，强制暂停并归纳
5. 一次只改一个变量
6. 已验证的可行路径禁止放弃

### 快速自检（每次操作后）

```
1. 我的上一次操作是：诊断 / 试探 / 修复？
2. 如果是试探 → 违反了规则 2，回到 STOP
3. 我的目标还在吗？我偏离了吗？
```

## 安装

### 一行命令

```bash
curl -o ~/.claude/skills/stop-research-act/SKILL.md --create-dirs https://raw.githubusercontent.com/shineSnow/stop-research-act/main/SKILL.md
```

然后在 `~/.claude/CLAUDE.md` 中添加：

```markdown
## 问题解决 SOP

遇到任何失败或错误，必须先诊断再行动。完整指南: /stop-research-act
```
